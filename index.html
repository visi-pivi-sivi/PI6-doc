Welcome to the instruction of PI6.

PI6 which is short for PI-VI, which is the middle name of VISI-PIVI-SIVI(Virtual Private Server). Actually, it is not a Server, it is born for making the maintenance of virtual private server simple and easier. PI6 is like a note book, recording every step that the author take while setting up the private server. But PI6 is NOT ONLY a note, it is executable.

For Example

If you want to set up a ssh protecting on your server. You can download the PI6 project, and cd to it directory, then execute:

bash setupssh

The first step in record is set the ssh connection keeping alive. So you will be asked whether to set it or not.

set the ssh connection keep alive.[y/n]

usually you have two choice, yes or no. If you put unexpected answer, do not be worry about it. You will be mention like:

press 'yes' or 'no'. for more info, try [s/e/m]:

the 's' is stand for 'source'. if you chose this here, you will see:

catIntoFile $'\n TCPKeepAlive yes\n ServerAliveInterval 120' /etc/ssh/ssh_config

the 'e' is stand for 'explanation'. if you chose this here, you will see:

set the ssh connection keeping alive.

the 'm' is stand for 'more'. if you chose this here, you will see:

To avoid to be disconnected, if the client not active for a short time.

After you made a decision, the progress will carry on step by step like those above, until the ssh setting up is done. You can use Ctrl + C to end the progress anytime you like.

PS. Currently also the setting is for Ubuntu. If you find there is any thing can improve, please feel free to let us know.
